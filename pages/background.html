Background

Our project is presented by Bosch, which is a famous german industrial company focus on advanced technologies. Bosch works in "Industry 4.0" and each of he devices they work on is represneted in a dynamic, graph-based database in RDF format. And we keep in contact with client by Skype for Business with Daniel Ewert(Bosch) and Steffen Stadtm√ºller(Bosch) to discuss our project requirements and give us resources.

Challenges:

Our project aims us to work out a website GUI which will communicate actions and manipulations to an underlying REST API which they have built for us. This API sends SPARQL queries to their RDF database. What the database sends back is converted to JSON. We have to visualise their graph based database, with attributes and relationships, visually so that it becomes easier for developers to understand the database and deduce patterns and relationships. However, RDF triples and the related query language SPARQL are not easily accessible for regular developers.

Framework for Success
  • 1. Teamwork
  • 2. Deeply understand how does REST API works.
  • 3. Good communication with client.
  • 4. Be familiar with RDF database
