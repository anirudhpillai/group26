RDF

Resource Descriptive Framework (RDF), is a data model and is the W3C standard for data interchange on the web. It is a way of organising data in graph form.
Data is stored in the form of triples. A triple is a statement of the form subject-predicate-object.
RDF also has multiple serialization formats. The way a particular triple is is encoded, depends on the format. Some of the available formats are: Turtle, N-Triples, RDF/XML. RDF/XML was the initial standard but users now prefer to use something like Turtle which is more human friendly.
A collection of these triples then represent a directed graph. RDF data can be stored in both relational databases and Triplestores.

Stack

We have to build a web based GUI so we had to use Javascript, HTML and CSS. We also looked at Elm but decided to stick with ES6 Javascript as we were familiar with it and had already built projects with it.

React

We decided to choose React to build our UI as the Component system which React is based on, will allow us to reuse most of our code. As the GUI will have similar looking pages for each node, we can use our components to handle these views.

Redux

We will be using Redux as our application architecture framework Redux will help us write maintainable code and make our app very scalable. Using Redux will help us test our app easily as it is based on writing pure functions, which will make things easy to reason about. It also provides us with Time travelling debugger which works based on the state tree employed in Redux.

Research on existing implementations and alternatives

What is neo4j?
Neo4j is a NoSQL graph database. The data is stored in nodes, that can have some set of properties, and links between those nodes, that describe how they relate to each other.

Why we looked at neo4j?
We used neo4j because we wanted to have a better understanding of how graph databases differed from traditional relational databases (mySQL/SQL). Neo4j also has the biggest community of developers among graph database systems, which means getting help is easy.

How we learned
Neo4j has an in-browser user interface that one can use to query database with a special language, Cypher (the traditional equivalent of the Cypher language is SQL). It shows the query results graphically. We learnt how to use Neo4j with the help of this user interface, and the tutorials and documentation available on the official website.

How it helped us for this project
It helped us a lot to make sense of what is involved in this project. Having experience with a graph database helped us grasp the challenges faced by our application. The fact that we could actually see the results of our queries gave us a deeper understanding of what can be achieved with graphs. It was a friendly introduction to graph databases before moving on to RDF/SPARQL tutorials.
