Stack

What choices were there and what are their strengths and weaknesses? We have to build a web based GUI so we had to use Javascript, HTML and CSS. We also looked at Elm but decided to stick with ES6 Javascript as we were familiar with it and had already built projects with it.

React

We decided to choose React to build our UI as the Component system which React is based on, will allow us to reuse most of our code. As the GUI will have similar looking pages for each node, we can use our components to handle these views.
We looked at Angular as well but ..... React also works better with Redux.

Redux

We will be using Redux as our application architecture framework Redux will help us write maintainable code and make our app very scalable. Using Redux will help us test our app easily as it is based on writing pure functions, which will make things easy to reason about. It also provides us with Time travelling debugger which works based on the state tree employed in Redux.

Tool Chain

  • ES6 with Babel: This will give us access to new and advanced features in Javascript
  • Gulp: for transpiling ES6
  • Eslint: for catching errors in our code and so that everyone on the team follows the recommended (Airbnb) style guide for ES6
  • Webpack: for bundling our dependencies
  • Travis CI: as a continuous integration tool which will run our tests whenever we push our code to Git

We only have to build a front end and don't have to write a server. This means we don't have to use continous deployment, but we are still using Travis CI as it will automatically run our tests and let us know if it is safe to merge our changes from our development branch to the master branch.
